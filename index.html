Raw RTCM Data Service

Wholesale GNSS reference station data streams to build your custom RTK/PPP positioning engines.

Decentralized GNSS Infrastructure

The **Global Earth Observation Decentralized Network (GEODNET)** is the world's largest blockchain-based GNSS reference station network. By deploying triple-frequency, multi-constellation base stations globally, GEODNET collects high-fidelity space weather and satellite observations in real-time.

The **Raw RTCM Data Service** is a wholesale data stream service designed for service providers, OEMs, and researchers. Instead of simple point corrections, this service provides direct, continuous 1 Hz raw observation streams (RTCM 3.2 MSM4 format) from a customized pool of stations, enabling you to build independent **Network RTK (VRS)**, **Precise Point Positioning (PPP)**, and **Precise Orbit Determination (POD)** engines.

Offline Stream Behavior

Once a connection is established for a specific base station mountpoint, the GEODNET server keeps the TCP socket connection active. In the event that a base station goes offline physically, the server will transmit a text message **"Mountpoint offline"** inside the payload.

When the base station recovers internet connectivity and starts logging data again, the observation data stream will resume automatically on the same connection.

Quality Control & Base Station Management

Understand quality control metrics, installation verification, and operational state variables.

Base Station States

Not all physical stations are active in the correction service caster. GEODNET algorithms filter signals based on continuous quality control checks. Station statuses include:

Active
Active Station has passed all installation check metrics, is streaming live data, and is fully integrated into the RTK/VRS/PPP service caster.
Online
Online Station is transmitting observation data packets to the cloud platform, but is currently excluded from active RTK calculation (have not passed the QC validation).
Offline
Offline Station has lost internet connectivity, or has power issues, and is currently not streaming any observation data to the backend.

GNSS QC Database Fields

GEODNET runs hourly quality checks on RTCM logs using TEQC and RTKLIB software. The resulting database contains the following metrics to evaluate station health:

Field Full Name Ideal Metric Target
Availability Data Availability (%) > 99.9%
MP12 Multipath on L1/L2 (m) < 0.5 m (Lower is better)
MP21 Multipath on L2/L1 (m) < 0.5 m (Lower is better)
IOD slips Ionospheric Delay Slips Lower values indicate high signal stability
o/slps Epochs per Cycle Slip Higher values indicate less atmospheric noise

Base Station Installation Quality Control Checks

To qualify for the **ACTIVE** state in the wholesale reference stream caster, base stations must maintain the following operational parameters:

  • Sky Visibility: Antenna must be mounted with clear 360° open-sky visibility to track low elevation satellites. Tracking less than 30 satellites simultaneously is flagged.
  • SNR Checks: The average Carrier-to-Noise density ratio (C/N0 or SNR) of tracked satellites must be above 32 dB-Hz. Satellite signals below 32 dB-Hz are filtered out.
  • Mechanical Stability: The physical structure must resist movement. Continuous PPP monitoring (NRCAN) detects if a station shifts. Movement of more than 5mm in horizontal coordinates will trigger automatic suspension from ACTIVE status.
